# Homepage Copy Review Checklist Skill

## Purpose

Review homepage copy for clarity, authority, reader value, and layout usefulness.

## Review Passes

### 1. Authority Pass

Check whether the first screen contains proof.

Good proof:

- production system
- adoption by senior audience
- recognized company/university
- measurable result
- rare achievement

### 2. Reader Value Pass

Replace self-centered phrasing with reader-centered phrasing.

Weak:

```text
Things I built.
```

Better:

```text
Tools you can use.
```

Weak:

```text
My blog.
```

Better:

```text
Guides from real accomplishments.
```

### 3. Specificity Pass

Remove vague phrases:

- "real stuff"
- "various systems"
- "practical solutions"
- "proof you can use"
- "interesting projects"

Replace with concrete objects:

- production agents
- setup guides
- automation workflows
- operational data
- reusable tooling

### 4. Layout Pass

Ask:

- Are important items visible immediately?
- Are guide/tool titles listed directly?
- Is the page too narrow?
- Are dense sections forced into two columns when one column is clearer?
- Is the reader forced to click before understanding value?

### 5. Proof Pass

For track record sections:

- Add context paragraph for company/university authority.
- Keep bullets result-focused.
- Use logos only if reliable.
- Remove broken links until assets exist.

### 6. Tone Pass

Avoid:

- bragging without utility
- negative comparison
- vague humility
- invented authority
- portfolio cliches

Prefer:

- proof-backed usefulness
- directness
- compact context
- concrete reader benefit

## Final Homepage Questions

- What can the reader learn?
- What can the reader use?
- Why should the reader trust this?
- How can the reader reach out?
- Is anything important hidden behind unnecessary clicks?
