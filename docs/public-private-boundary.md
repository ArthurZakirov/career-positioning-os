# Public/Private Boundary

Career Positioning OS is meant to be public methodology. Real evidence belongs in a private repo.

## Public Repo May Contain

- reusable methods
- schemas
- public-safe templates
- dummy examples
- generic skills
- sanitized diagrams
- install and usage docs

## Private Repo Should Contain

- real achievements
- raw notes
- employer details
- internal links
- exact metrics not approved for publication
- target roles and job strategy
- applications and recruiter messages
- compensation, performance review, or negotiation context

## Workflow

Use this flow:

```text
private raw notes
  -> private achievement packet
  -> sanitized public abstraction
  -> public artifact
```

Do not skip directly from raw private notes to public output.

## Dummy Data Rule

Examples in this repo must be fictional or sanitized enough that they cannot reveal a real employer, person, internal system, customer, private metric, or job-search strategy.

If an example comes from real work, rewrite it as a generic pattern and remove the original details.

## Output Rule

This repo can define how to generate resume bullets, LinkedIn sections, portfolio case studies, and interview stories. It should not contain one person's real final outputs unless those outputs are explicitly approved for publication.
