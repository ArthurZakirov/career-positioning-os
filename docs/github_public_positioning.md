# GitHub Public Positioning

This repository includes a GitHub-positioning bundle alongside the career-writing skills.

## Included Skills

- `github-repo-product-naming` for naming repos, products, CLIs, plugins, and skill packs with the right association.
- `github-repo-launch-page` for public repo README structure, launch-page polish, and GitHub metadata.
- `github-profile-authority-page` for GitHub profile positioning, pinned repos, and credibility framing.

## Ecosystem Names

Use these names as the default product family for Arthur's public GitHub ecosystem:

- `ProofStack` -> career proof and public-safe positioning
- `AgentDesk` -> machine and AI-agent workstation setup
- `SystemSmith` -> engineering methods for code, agents, and architecture

## How To Use

1. Use `github-repo-product-naming` before naming or renaming a public repo.
2. Use `github-repo-launch-page` when turning a repo into a shareable project page.
3. Use `github-profile-authority-page` when turning the GitHub profile into a credible landing page.
4. Keep the product names consistent across repo slug, README title, About description, and pinned repositories when possible.
